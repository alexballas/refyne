Refyne Vendoring Gap Analysis

Generated: 2026-07-02 Refyne: 1bf007f Fyne upstream: develop@4a089fc go-gl: v3.4/glfw/v0.1.0-pre.1 GLFW: 3.4

Executive Summary

Refyne vendors the go-gl GLFW binding and the GLFW C source under internal/glfw. This gives Refyne direct control over Wayland behavior, custom decorations, resize timing, cursor protocols, icon protocols, and portal drops, but it creates a local maintenance fork.

A separate upstream setup, using Fyne plus go-gl plus GLFW as independent dependencies, reduces vendored-code ownership and tracks upstream packaging more cleanly. The main gap is that the separate setup does not include Refyne's local Wayland and portal patches unless they are upstreamed or carried as an external patch set.

Baseline Note

GitHub does not currently expose a Fyne devel branch. The default upstream development branch is develop, so this report uses fyne-io/fyne develop at 4a089fc from 2026-07-01. The branch changelog contains a 2.8.0 entry dated 2026-07-05, which is after this report date.

Separate Upstream Setup

Fyne: Features or Changes Added

  • New canvas objects and render paths: ArbitraryPolygon, BezierCurve, Blur, Ellipse, Shader, shader animation, and hardware-accelerated shadows.
  • Scheduled notifications, richer Form validation, new RichText/Markdown table and checklist support, underline and strikethrough text.
  • Accessibility support behind -tags accessibility, secondary monitor support, cache APIs, window position and always-on-top APIs.
  • Wayland is selected by default at runtime where available; Fyne develop also updates its module dependency to a newer go-gl GLFW 3.4 pseudo-version.
  • Behavior changes include popup padding removal, Polygon renamed to RegularPolygon, macOS scrollbar preference following, and launch warnings for apps not using fyne.Do.

go-gl: Features or Changes Added

  • Adds a GLFW 3.4 Go module at github.com/go-gl/glfw/v3.4/glfw, embedding GLFW 3.4 C sources.
  • Exposes GLFW 3.4 platform APIs such as GetPlatform, PlatformSupported, PlatformHint, and platform constants for Win32, Cocoa, Wayland, X11, and Null.
  • Adds Go access to InitAllocator, GetWindowTitle, captured cursor mode, mouse passthrough, initial position hints, framebuffer scaling, ANGLE hints, Wayland app_id, and libdecor hints.
  • Adds constants for newer standard cursors and GLFW 3.4 error codes.
  • Does not include Refyne's wl_refyne.c, portal drop support, icon protocol bindings, or custom resize/shadow behavior.

GLFW: Features or Changes Added

  • Runtime platform selection across compiled backends via GLFW_PLATFORM, glfwGetPlatform, and glfwPlatformSupported.
  • More standard cursor shapes, mouse event passthrough, window title query, captured cursor mode, and custom heap allocator support.
  • Framebuffer scaling and initial window-position hints, ANGLE backend selection, Win32 keyboard menu and startup show hints, and Cocoa NSView native access.
  • Wayland improvements include libdecor decorations and a GLFW_WAYLAND_APP_ID hint.
  • X11 gains a Vulkan surface-selection hint; GLFW 3.4 also changes version-string assumptions because one binary may include multiple native backends.
